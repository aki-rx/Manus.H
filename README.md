# Open WebUI — Pre-Configured Deployment

A ready-to-deploy [Open WebUI](https://github.com/open-webui/open-webui) setup with **Docker Compose**, pre-configured for **OpenRouter** and **Ollama**, featuring a custom terminal-styled theme, provider-colored model badges, and automated model renaming.

**API keys and secrets are never stored on disk.** They are passed as terminal environment variables at runtime and exist only in memory.

---

## What's Included

| Component | Description |
|---|---|
| **Docker Compose** | Single-command deployment of Open WebUI + Ollama |
| **OpenRouter** | Pre-wired to `https://openrouter.ai/api/v1` with auto-routing model |
| **Ollama** | Local model server running alongside Open WebUI |
| **Custom Theme** | JetBrains Mono font, yellowish accent color |
| **Provider Colors** | Model entries color-coded by provider (OpenAI, Anthropic, Google) |
| **Faded Params** | Parameter sizes (e.g. 70B) shown faded next to model names |
| **Admin Account** | Auto-created on first launch via environment variables |
| **Model Renamer** | Script to rename models to a clean display format |
| **Auto-Routing** | Default model set to `openrouter/auto` for intelligent selection |
| **Secure Secrets** | API keys passed at runtime only, never written to disk |

## Quick Start

### Prerequisites

Docker and Docker Compose must be installed on your server. That's it.

### 1. Clone

```bash
git clone https://github.com/aki-rx/Manus.H.git
cd Manus.H
```

### 2. Launch (Recommended)

Use the interactive launcher — it prompts for your secrets and keeps them in memory only:

```bash
bash start.sh
```

The script will ask for:
- Your **OpenRouter API key**
- Your **admin password**
- It auto-generates a **secret key** for JWT tokens

Nothing is saved to disk. When you close the terminal, the secrets are gone.

### 3. Launch (Manual)

If you prefer to export variables yourself:

```bash
export OPENAI_API_KEY="sk-or-your-openrouter-key"
export WEBUI_SECRET_KEY="$(openssl rand -hex 32)"
export WEBUI_ADMIN_PASSWORD="your-strong-password"

docker compose build
docker compose up -d
```

Open WebUI will be available at **http://localhost:3000**.

### 4. Rename Models (Optional)

After Open WebUI is running and models are loaded:

```bash
pip install requests
python3 scripts/rename_models.py --wait 30
```

This converts raw model IDs like `meta-llama/llama-3.1-70b-instruct` into clean names like **Llama 3.1** with a faded **70B** parameter badge.

Preview changes without applying:

```bash
python3 scripts/rename_models.py --dry-run
```

## How Secrets Work

Secrets are handled through **shell environment variables** — they exist only in your terminal session's memory and are passed to Docker at runtime.

| Secret | How It's Set | Stored On Disk? |
|---|---|---|
| `OPENAI_API_KEY` | Prompted by `start.sh` or `export` in terminal | No |
| `WEBUI_SECRET_KEY` | Auto-generated by `start.sh` or `export` in terminal | No |
| `WEBUI_ADMIN_PASSWORD` | Prompted by `start.sh` or `export` in terminal | No |

Non-secret defaults (UI name, admin email, etc.) can optionally be stored in `.env.example` and are safe to commit.

Docker Compose reads the exported variables from your shell via `${VAR}` syntax. If a required secret is missing, it will fail with a clear error message instead of starting with empty keys.

## Custom Theme

### JetBrains Mono Font

All UI text uses [JetBrains Mono](https://www.jetbrains.com/mono/), a monospaced typeface designed for developers. It loads from Google Fonts and applies globally across the interface, including chat messages, code blocks, and the sidebar.

### Yellowish Accent Color

Interactive elements use a warm yellow (`#f5c518`) as the accent color. This includes the send button, toggle switches, focus rings, scrollbar thumbs, tooltips, and active sidebar items.

### Faded Model Parameters

Model names display the parameter count (e.g., `70B`) in a faded, lighter style next to the model name. This keeps the interface clean while still showing model size at a glance.

## Provider Background Colors

Model entries in the selector are automatically color-coded by provider:

| Provider | Color | Detection Keywords |
|---|---|---|
| **OpenAI / ChatGPT** | Greyish white background | `gpt`, `chatgpt`, `openai`, `o1`, `o3`, `o4` |
| **Anthropic / Claude** | Warm orange tint | `claude`, `anthropic` |
| **Google / Gemini** | Blue-purple-red gradient | `gemini`, `google`, `palm` |
| **Others** | No background color | Everything else |

The colors are subtle and semi-transparent, working well in both light and dark modes.

## Model Naming Format

The renamer script converts model IDs into human-readable names:

| Raw Model ID | Display Name |
|---|---|
| `meta-llama/llama-3.1-70b-instruct` | Llama 3.1 *70B* |
| `mistral:7b` | Mistral *7B* |
| `anthropic/claude-3.5-sonnet` | Claude 3.5 Sonnet |
| `openai/gpt-4o` | Gpt 4o |
| `google/gemini-2.0-flash` | Gemini 2.0 Flash |
| `openrouter/auto` | Auto Router |
| `deepseek/deepseek-r1:32b` | Deepseek R1 *32B* |

The italicized parameter sizes appear faded in the actual UI.

## Configuration Reference

Non-secret settings can be overridden by exporting them before running:

| Variable | Description | Default |
|---|---|---|
| `WEBUI_NAME` | Name shown in the UI header | `AI Chat` |
| `WEBUI_ADMIN_EMAIL` | Admin account email | `admin@example.com` |
| `WEBUI_ADMIN_NAME` | Admin display name | `Admin` |
| `OPENAI_API_BASE_URL` | OpenRouter endpoint | `https://openrouter.ai/api/v1` |
| `DEFAULT_MODELS` | Default model for new chats | `openrouter/auto` |
| `TASK_MODEL_EXTERNAL` | Model for title/tag generation | `openrouter/auto` |
| `ENABLE_SIGNUP` | Allow new user registration | `false` |

For the full list, see the [Open WebUI documentation](https://docs.openwebui.com/reference/env-configuration/).

## GPU Support (Ollama)

Uncomment the `deploy` section in `docker-compose.yml`:

```yaml
ollama:
  image: ollama/ollama:latest
  deploy:
    resources:
      reservations:
        devices:
          - driver: nvidia
            count: all
            capabilities: [gpu]
```

Requires the [NVIDIA Container Toolkit](https://docs.nvidia.com/datacenter/cloud-native/container-toolkit/install-guide.html).

## Troubleshooting

**"Set OPENAI_API_KEY in your terminal before running":** Export your key first: `export OPENAI_API_KEY="sk-or-..."` or use `bash start.sh`.

**Models not loading from OpenRouter:** Verify your API key is valid. Check logs with `docker compose logs open-webui`.

**Provider colors not showing:** Run the model renamer, or the colors will apply based on raw model IDs containing provider keywords.

**Custom CSS not applied:** Rebuild with `docker compose build --no-cache` and restart.
